What is a Lean Startup?

From Wikipedia:

Lean Startup is a set of processes used by entrepreneurs to develop products and markets, combining Agile Software Development, Customer Development and existing software platforms (usually FOSS).

Customer Development

A model for early stage companies, pioneered by Steven Gary Blank, a serial Silicon Valley entreprenuer.

FOSS

Free and open source software.

(Ruby, Rails, Linux, MySQL, Apache, Sphinx)

The Agile Manifesto (2001)

  • Individuals and interactions over processes and tools
  • Working software over comprehensive documentation
  • Customer collaboration over contract negotiation
  • Responding to change over following a plan

What does this mean?

  • Embrace change
  • Iterate rapidly
  • Just do it

What does this NOT mean?

  • Winging it
  • Cowboy programming

Agile tools and processes

  • User Stories
  • Version Control
  • Stand-ups and retrospectives
  • Automated Testing
  • Continuous Integration

Agile tools and processes

  • User Stories - progression on the Kanban
  • Version Control - git and github
  • Stand-ups and retrospectives
  • Automated Testing - Cucumber, RSpec, Jasmine
  • Continuous Integration - Hudson

User story

  • Written by customers
  • Using business language

More than a feature request

  • For whom? - Who actually needs it?
  • Why? - Where's the business value?

User story format

Feature: <Short description of the feature.>

As a <role>

I want <goal/desire>

So that <benefit>

Feature: Show tribes on the dashboard

As a logged in user

I want to see my tribes on the dashboard

So that I can click on them and see what's new

Feature: Shopping revenue report

As the Tribesports accountant

I want a report for this week's purchases

So that I can see how much money we made

Feature: Move product images to S3

As the Tribesports tech department

I want all the images of products to be stored on Amazon S3

So they load more quickly and save space

The Kanban - Icebox

Where a story starts.
Might never actually be implemented.

The Kanban - Backlog

Stories that are on their way to be developed.

The Kanban - Selected

Selected to be developed next.
The only ones developers can choose to work on.

The Kanban - In Progress

Stories currently being worked on by the development team.

The Kanban - Done

Stories that have been implemented,
but aren't live yet.

The Kanban - Deployed

Stories that have been released to the
production website, tribesports.com
